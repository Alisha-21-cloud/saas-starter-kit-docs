Enterprise SaaS Starter Kit

The open-source Next.js boilerplate that handles everything your SaaS app needs out of the box — authentication, teams, billing, audit logs, and more — so you can focus on what makes your product unique.

Open Source Next.js 14 TypeScript Apache 2.0

Saves you months of development by starting you off with all the features that are the same in every SaaS product — multi-tenancy, auth, payments, webhooks, audit logs — so you can build what actually matters.

Please star the repo if you want to help continue developing and improving the SaaS Starter Kit! It motivates us to keep shipping.

Getting Started

Installation

Follow these steps to get a local copy of the project up and running.

01
Fork and clone the repository
Start by forking the repo on GitHub, then clone your fork locally.
bash 
git clone https://github.com/<your_github_username>/saas-starter-kit.git
02
Go to the project folder
bash 
cd saas-starter-kit
03
Install dependencies
bash 
npm install
04
Set up your .env file
Duplicate the example environment file and fill in your values.
bash 
cp .env.example .env
05
Create a database (Optional — Docker)
We provide a docker-compose.yml with a PostgreSQL container to make setup easier.
bash 
docker-compose up -d
06
Set up database schema
Push the Prisma schema to your database.
bash 
npx prisma db push
Getting Started

Running the App

Start the Development Server

bash 
npm run dev

Open Prisma Studio

Prisma Studio is a visual editor for the data in your database. A great way to inspect and manage records during development.

bash 
npx prisma studio

Running E2E Tests

The project uses Playwright to execute end-to-end tests. All tests live in the /tests folder. The HTML test report is generated inside the report folder. Supported browsers: Chromium and Firefox.

bash 
# Install Playwright browser binaries
npm run playwright:update

# Run the E2E test suite
npm run test:e2e
Update playwright.config.ts in the project root to customize the Playwright configuration — change base URL, timeout, browser targets, and more.

Feature Configuration

Authentication

Authentication is powered by NextAuth.js. The default login options are email (magic link) and GitHub OAuth. Configuration lives in pages/api/auth/[...nextauth].ts.

">

NextAuth.js Setup

  • Generate a secret key and add it to .env as NEXTAUTH_SECRET
  • Configure SMTP_* variables for magic link login emails (AWS SES, SendGrid, or Resend)
  • For GitHub OAuth: create an OAuth App in the GitHub developer console, add GITHUB_CLIENT_ID and GITHUB_CLIENT_SECRET
  • For Google OAuth: create an OAuth App in the Google Cloud Console, add GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET
bash 
# Generate a secure secret for NextAuth.js
openssl rand -base64 32
.env 
NEXTAUTH_SECRET=your-generated-secret
NEXTAUTH_URL=http://localhost:3000

# SMTP for magic link emails
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=user@example.com
SMTP_PASSWORD=your-password
SMTP_FROM=noreply@yourapp.com

# GitHub OAuth
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret

# Google OAuth
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
Feature Configuration

Svix Webhooks

Webhooks are orchestrated by Svix. Events are emitted on user and team CRUD operations, which can be consumed by your own services or third-party integrations.

">

Svix Setup

  • Create an account at svix.com
  • Copy your authentication token from the Svix dashboard
  • Add SVIX_API_KEY to your .env file
.env 
SVIX_API_KEY=your-svix-api-key
Feature Configuration

Stripe Payments

Payments are handled by Stripe, including subscription management and billing. The webhook endpoint is at /api/webhooks/stripe.

">

Stripe Setup

  • Create a Stripe account at stripe.com
  • Copy the API secret key from the Stripe Dashboard → add as STRIPE_SECRET_KEY
  • Create a webhook endpoint in the Stripe Dashboard pointing to your-app.com/api/webhooks/stripe
  • For local development, use the Stripe CLI forwarder to forward events to localhost
  • Copy the webhook signing secret → add as STRIPE_WEBHOOK_SECRET
.env 
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
Feature Configuration

Recaptcha

Protect your forms and API endpoints with Google reCAPTCHA Enterprise.

">

reCAPTCHA Setup

  • Create an account at Google reCAPTCHA (creates a Google Cloud account if needed)
  • From the Google Cloud Console, copy the reCAPTCHA ID → add as RECAPTCHA_SITE_KEY
  • Click Key Details → Integration → Use legacy key to get the secret key → add as RECAPTCHA_SECRET_KEY
.env 
RECAPTCHA_SITE_KEY=your-recaptcha-site-key
RECAPTCHA_SECRET_KEY=your-recaptcha-secret-key
Feature Configuration

Sentry Error Tracking

Monitor errors and performance issues in production with Sentry.

">

Sentry Setup

  • Create an account at sentry.io, skip onboarding, and create a new Next.js project
  • At the bottom of the setup page, copy the DSN string → add as SENTRY_DSN
  • Other Sentry environment variables are optional
.env 
SENTRY_DSN=https://xxxx@o0.ingest.sentry.io/0

Deploy to Heroku →

Writing Good Bug Reports

Please try to create bug reports that are:

  • Reproducible — include steps to reproduce the problem consistently
  • Specific — include as much detail as possible: version, environment, OS
  • Unique — do not duplicate existing open issues; search first
